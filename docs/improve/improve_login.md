* **Thông điệp/branding bên trái chưa “match” sản phẩm**: “Your pathway to results begins here” khá generic, không nói bạn là tool AI capture task đa nền tảng → đổi headline thành value cụ thể (vd: “Capture tasks from email & chat, automatically”), thêm 1 dòng phụ ngắn về Gmail/Chat/Slack… để user hiểu ngay trước khi login; giữ hình nhưng giảm tính “stock/abstract” hoặc thêm overlay gradient + logo/tagline để đồng bộ brand, tránh cảm giác landing khác app
* **Bố cục thị giác chưa cân bằng**: khối ảnh trái quá nặng, form phải hơi “lạc lõng” (nhiều khoảng trắng) → tăng trọng lượng form bằng card nhẹ (border/ shadow rất soft), hoặc thu nhỏ ảnh/giảm saturation/blur nhẹ để form là focal point; canh vertical center form tốt hơn (hiện hơi cao) để tạo cảm giác ổn định
* **CTA “Log in” đang là dạng outline nên thiếu ưu tiên**: nút chính nên là solid (primary fill) + hover/pressed rõ, và state disabled/loading chuẩn; outline có thể dùng cho secondary (ví dụ “Sign up”) để hierarchy đúng
* **Thiếu các nhánh đăng nhập phổ biến cho SaaS/B2B**: chưa thấy “Continue with Google/Microsoft” (đặc biệt bạn tích hợp Google Chat/Gmail) → thêm SSO buttons ở trên form (Google Workspace/Microsoft), giảm friction và tăng trust; nếu chưa làm SSO thì tối thiểu thêm “Sign in with Google” vì hợp ngữ cảnh sản phẩm
* **Input/email đang thiếu chuẩn microcopy & validation**: placeholder “sonkame” dễ gây hiểu nhầm (không giống email) → placeholder dạng “[name@company.com](mailto:name@company.com)”; thêm inline validation (sai format email, caps lock) và error copy rõ ràng; label nên consistent casing (“Email”, “Password” ok) nhưng có thể thêm helper text nhỏ nếu sản phẩm enterprise (“Use your work email”)
* **Password UX còn thiếu vài chi tiết chuẩn**: icon eye ok nhưng cần tooltip/aria label + trạng thái rõ; thêm “Forgot password?” ngay dưới password hoặc cạnh Remember Me; hỗ trợ Enter submit, focus ring rõ (accessibility)
* **“Remember Me” có thể gây rủi ro/không đúng kỳ vọng**: với web app thường “Keep me signed in” rõ nghĩa hơn; mặc định nên OFF (trừ khi nghiên cứu cho thấy user muốn), và nếu ON cần mô tả ngắn (vd: “on this device”)
* **Link Sign up đang yếu về hierarchy & copy**: “Don’t have an account? Sign up” ổn nhưng nên tăng affordance (màu/link hover/underline) và cân nhắc đặt gần CTA hoặc ngay dưới; nếu sản phẩm có waitlist/approval, copy nên phản ánh (“Request access” thay vì “Sign up”) để giảm hụt hẫng
* **Thiếu trust cues cơ bản cho màn login**: thêm 1 dòng nhỏ dưới CTA kiểu “We never post without permission” hoặc “Secure OAuth with Google Workspace” (nếu đúng), hoặc icon lock + “Encrypted” để tăng cảm giác an toàn; có thể thêm link “Privacy/Terms” ở footer (nhỏ)
* **Khả năng đọc trên nền ảnh**: text trắng bên trái ổn nhưng cần kiểm tra contrast ở mọi màn; nên có overlay gradient cố định để đảm bảo WCAG và tránh vùng sáng sau chữ (hiện ổn nhưng nên khóa bằng design token)
* **Consistency với design system**: radius/outline của input và button đang hơi “đơn giản” so với ngôn ngữ UI cards của app (nhiều bo tròn/soft shadow) → đồng bộ radius (ví dụ 12–16), shadow và spacing theo 8pt grid; thống nhất style icon (stroke, size) và màu primary
* **States còn thiếu**: thêm loading state khi login (spinner + disable fields), error state (wrong password), network offline, rate-limit; success redirect nên có transitional feedback để user không thấy “nhảy” đột ngột

Ok, chưa có Google OAuth nhưng sẽ có → mình sẽ thiết kế UI theo kiểu **progressive enhancement**: hiện tại chạy email/password ngon, nhưng cấu trúc sẵn để bật Google OAuth sau này mà không phải đập UI.

* **Flow tổng (progressive)**: V1 chỉ email/password; vẫn chừa slot SSO ở trên nhưng đặt ở trạng thái “Coming soon”/disabled (không làm user hụt hẫng) hoặc ẩn hoàn toàn bằng feature flag; khi OAuth xong chỉ bật lại đúng vị trí đó, không đổi layout
* **SSO placeholder (khuyến nghị)**: thay vì show nút “Continue with Google” bị disable (dễ gây bực), tốt hơn là **hiển thị dạng text hint** dưới form: “Google sign-in coming soon” + link “Notify me” (optional) hoặc “Use your work email for now”; nếu bạn muốn show button thì làm disabled kèm tooltip “Available soon” (nhưng chỉ nên khi bạn chắc 1–2 tuần bật)
* **Form spec V1 (email/password) chuẩn SaaS**: header “Welcome back” + sub “Sign in to SpiderX to continue”; Email label “Work email”, placeholder “[name@company.com](mailto:name@company.com)”, validation inline; Password label + “Forgot password?”; Remember me đổi thành “Keep me signed in (on this device)” và default OFF
* **CTA hierarchy (sửa ngay)**: nút primary **solid** full-width “Log in”; states: disabled khi thiếu input; loading “Logging in…” + spinner; error toast nhẹ + inline error dưới field; success redirect có transition mượt
* **Copy “Sign up” tùy mô hình**: nếu chưa mở đăng ký rộng thì đổi “Sign up” → “Request access”; nếu mở thì giữ “Create an account”; tránh để user bấm rồi gặp dead-end
* **Trust & policy**: thêm 1 dòng nhỏ dưới CTA: “Secure sign-in. We never post without permission.” (chỉ dùng nếu đúng); footer “Privacy • Terms” để tăng tin tưởng
* **Chuẩn bị cho OAuth sau này (không vỡ UI)**: ngay khi có Google OAuth, thêm block SSO ở trên: Button “Continue with Google” (solid/white), divider “or”, rồi mới email/password; copy phụ: “Recommended for Google Workspace users” (vì bạn tích hợp Gmail/Chat)
* **Accessibility & consistency**: focus ring rõ, Enter submit, aria labels cho eye icon; chuẩn hoá radius/spacing theo design system app (vd radius 16, spacing 8pt), tránh nút outline cho primary

Nếu bạn nói luôn giúp mình **signup hiện tại là mở hay request access**, mình sẽ viết ra **bản copy cuối cùng (từng dòng text trên UI)** theo đúng model của bạn để bạn paste vào code/Figma ngay.
